Here is the CSS strategy to tighten the visual language. We are pivoting to a **"Clean Brutalist"** aesthetic: flattening the geometry (as per your design system) but using high-contrast borders and shadows to keep the editorial edge.

### 1. The Tags (Unification)
*Problem:* The skew in Row 1 breaks the "square corners" rule of your system.
*Solution:* Flatten all tags into high-impact "stickers." We will use the Yellow fill for maximum contrast against the Ink/Forest elements.

```css
/* Apply to all .tag classes in Row 1, 2, and 3 */
.tag {
  background-color: #FFD600;      /* User Yellow */
  color: #0D0D0D;                 /* User Ink */
  border-radius: 0px;             /* Enforce square */
  border: none;                   /* Remove inconsistent borders */
  transform: none;                /* REMOVE Row 1 skew */
  padding: 0.4rem 0.8rem;
  font-family: inherit;           /* Or your mono font */
  font-weight: 700;
  text-transform: uppercase;
  letter-spacing: 0.05em;
  font-size: 0.85rem;
  display: inline-block;
}
```

### 2. The Descriptions (Typography)
*Problem:* Green borders in Row 1 create visual clutter.
*Solution:* Remove the box. Use the **Forest** color and negative space to define the hierarchy. This makes Row 1 and 3 text blocks feel more related to the clean typography of Row 2.

```css
/* Apply to description text containers */
.work-description {
  border: none;                   /* Remove green border */
  color: #275E48;                 /* User Forest */
  background: transparent;
  padding: 0;                     /* Remove box padding */
  margin-top: 1rem;
  line-height: 1.6;
  font-size: 1.125rem;
  max-width: 60ch;                /* Editorial readability standard */
}
```

### 3. Row Separators (Rhythm)
*Problem:* Inconsistent breaks disturb the reading flow.
*Solution:* Hard, magazine-style dividers using the **Ink** color.

```css
/* Apply to the container or bottom of Row 1 and 2 */
.row-separator {
  border-bottom: 2px solid #0D0D0D; /* High contrast divide */
  margin-bottom: 4rem;
  padding-bottom: 4rem;
}
```

### 4. Images & Hover Effects (The "Editorial" Character)
*Problem:* Tilted (R1) vs Parallax (R2) vs Clip-path (R3).
*Solution:* Standardize the *resting* state to straight rectangles. Unify the *hover* state to a "Hard Lift." This mimics the "floating" vibe of Row 2 but keeps the layout grid of Rows 1 & 3 disciplined.

```css
/* Base Image Style for Row 1 and 3 */
.work-image {
  border-radius: 0px;
  transform: none;                /* Remove Row 1 tilt */
  clip-path: none;                /* Remove Row 3 clip */
  filter: grayscale(100%);        /* Optional: High-fashion feel */
  transition: all 0.3s cubic-bezier(0.25, 0.46, 0.45, 0.94);
  border: 1px solid #0D0D0D;      /* Thin ink border definition */
}

/* Hover Unification for All Rows */
.work-image:hover {
  filter: grayscale(0%);          /* Color comes back on hover */
  transform: translate(-8px, -8px); /* The "Lift" */
  box-shadow: 8px 8px 0px #275E48;  /* User Forest - Hard Shadow */
}
```

### 5. Creative Director's Note (The Polish)
To integrate Row 2 (the huge text row) completely:
*   Ensure the floating image in Row 2 *also* receives the `box-shadow` and `border: 1px solid #0D0D0D` to tie it to the grid images.
*   Change the massive text color in Row 2 to **Ink (#0D0D0D)** but change the stroke/fill on hover to **Lavender (#C8B4FF)** to introduce that fourth color from your palette which is currently missing.

```css
/* Row 2 Big Text Hover Idea */
.row2-huge-text {
  color: #0D0D0D;
  transition: color 0.3s ease;
}
.row2-huge-text:hover {
  color: #C8B4FF; /* Lavender interaction */
}
```
